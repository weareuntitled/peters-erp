# Phase 999.2: Voice-to-JSON Offer Generator — Specification

**Created:** 2026-04-30
**Ambiguity score:** 0.02 (gate: ≤ 0.20)
**Requirements:** 5 locked

## Goal

Integrate a global voice-recording feature that transcribes speech locally and uses an LLM to generate structured document drafts (offers/invoices) directly in the database.

## Background

Currently, users must manually type all data for offers and invoices in the `QuoteCreationPage`. There is no way to ingest unstructured data (like a voice memo) into the system. This phase adds a background ingestion pipeline that allows a user to "speak" an offer into existence while moving around the dashboard or other pages.

## Requirements

1. **Global Voice Recorder UI**: A persistent floating action button (FAB) available on all dashboard-related pages.
   - Current: No voice recording interface exists.
   - Target: A red floating button in the bottom-right corner. One click starts recording; second click (or time limit) stops and sends. Visual feedback (pulsating ring) must be shown during recording.
   - Acceptance: Button is visible on Dashboard; clicking it toggles "recording" state; pulsating animation is active while recording. Button is hidden when LLM settings are not configured.

2. **MediaRecorder Integration**: Capture and upload audio using native browser APIs.
   - Current: No audio capture logic.
   - Target: Captures audio as `.webm` blob. Implements a hard limit of 5 minutes (300s). Automatically stops and submits `FormData` to `/api/voice-to-draft` when limit is reached.
   - Acceptance: Recording automatically terminates at 300s; `POST` request is observed in network tab with `file` field containing audio data.

3. **Local Transcription (faster-whisper)**: Backend processing of audio without cloud dependencies.
   - Current: Backend only handles text/JSON.
   - Target: `POST /api/voice-to-draft` endpoint receives file, saves to temp, and transcribes using `faster-whisper` (model: `base`).
   - Acceptance: Endpoint returns a successful 200 OK with the transcribed text in the response body during initial testing.

4. **LLM Mapping (Text to JSON)**: Convert unstructured text into document fields.
   - Current: No LLM integration.
   - Target: Use `openai` python library to send transcription to an LLM (e.g. local Ollama or Groq). The LLM must return JSON containing `kunde_suche` and an array of `positionen` (bezeichnung, menge).
   - Acceptance: Backend successfully parses LLM output into a Python dictionary matching the `Dokument` and `Position` schemas.

5. **Draft Management & Dashboard Integration**: Save results and show them to the user.
   - Current: Dashboard only shows "Angebote" and "Rechnungen".
   - Target: Create a `Dokument` record with `status="entwurf"`. Store original transcription in `bemerkung`. Update Dashboard toggle to include "Entwürfe" (Drafts) which filters the list to documents with the "entwurf" status.
   - Acceptance: After recording, a new row appears in the Dashboard list when the "Entwürfe" tab is selected.

## Boundaries

**In scope:**
- `VoiceRecorder.tsx` component (React, native MediaRecorder).
- Global FAB placement in `DashboardLayout` (visible only when LLM is configured).
- `POST /api/voice-to-draft` endpoint.
- `faster-whisper` integration (local processing).
- `openai` library setup for LLM mapping.
- New `entwurf` status for documents.
- "Entwürfe" tab/filter in `DashboardPage`.
- Settings UI for LLM API configuration (Base URL, Encrypted API Key, Model).
- Simple encryption layer for API key storage.

**Out of scope:**
- Frontend form for manual transcription editing (this happens in existing Edit pages).
- Voice commands for app navigation (this is data entry only).
- Multi-speaker diarization (assumes single user).
- Customizing the LLM prompt via UI (prompt is hardcoded for now).

## Constraints

- **Privacy**: No full customer/article lists sent to LLM. The LLM extracts names/terms; the backend performs local fuzzy matching.
- **Performance**: Faster-whisper must use the `base` model to ensure reasonable response times on CPU.
- **No External Libs (Frontend)**: Strictly use browser native `MediaRecorder` and `navigator.mediaDevices.getUserMedia`.
- **Default LLM**: Default `LLM_BASE_URL` set to `http://localhost:11434/v1` (Ollama).

## Acceptance Criteria

- [ ] Voice recorder button is visible globally (only when LLM settings configured).
- [ ] Recording stops automatically at 5 minutes.
- [ ] Pulsating red animation triggers during recording.
- [ ] Backend transcribes audio locally using faster-whisper.
- [ ] Backend creates a document with `status="entwurf"` in the database.
- [ ] Transcription text is saved in the `bemerkung` field of the draft.
- [ ] Dashboard toggle has "Entwürfe" option that shows these drafts.
- [ ] Settings page has an AI configuration section with Base URL, API Key (encrypted), and Model fields.
