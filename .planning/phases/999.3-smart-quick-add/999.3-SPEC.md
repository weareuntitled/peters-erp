# Phase 999.3: SmartCombobox + Quick-Add — Specification

**Created:** 2026-04-30
**Ambiguity score:** 0.05 (gate: ≤ 0.20)
**Requirements:** 9 locked

## Goal

Enable users to search, select, and quickly create customers and articles directly from within the QuoteCreationPage and other document forms. When a search yields no results, the user can create a new record inline — with smart duplicate detection and intelligent article recommendations based on purchase history.

## Background

Currently, the `QuoteCreationPage` uses separate `CustomerSearchCombobox` and `ArticleSearchCombobox` components that only search existing records. There is no way to create a new customer or article without leaving the page. This creates friction for users who want to quickly capture a new contact or product while drafting an offer/invoice.

## Requirements

1. **Smart Search with Debounce**: Search inputs debounce at 300ms to prevent flooding the backend.
   - Current: Every keystroke triggers a search request
   - Target: 300ms debounce applied to all search inputs
   - Acceptance: Typing "Müller" fires exactly 1 request after 300ms of inactivity

2. **Smart Duplicate Check**: Before creating a new customer or article, the backend checks if a similar record already exists.
   - Current: No duplicate check — duplicate records can be created
   - Target: On Quick-Add attempt, backend checks for similar names. Returns `409 Conflict` if duplicate found. Frontend shows warning: "This already exists. Select instead?"
   - Acceptance: Creating "Müller Bau" when "Müller" exists → warning dialog appears

3. **Article Quick-Add with Warengruppe Chips**: When no article is found, a "+ {search_term}" button appears that opens an inline form with Warengruppe category chips.
   - Current: No inline article creation
   - Target: Chips for Material, Werkzeug, Dienstleistung (etc.) appear in the modal. User selects a chip and saves.
   - Acceptance: User types "Kupfer Platte" → no results → "+ Kupfer Platte" button → clicks → chips appear → user selects "Material" → saves → article created with that Warengruppe

4. **Customer Quick-Add (Minimalist 1-Step)**: When no customer is found, a "+ {search_term}" button appears. Clicking it saves immediately with just the name.
   - Current: Must navigate to separate page to create customer
   - Target: Name-only creation via Quick-Add modal
   - Acceptance: User types "Müller GmbH" → no results → "+ Müller GmbH" button → clicks → customer created

5. **Price History Insights (Backend)**: Endpoint `/api/artikel/{id}/insights` returns historical pricing and co-purchase recommendations.
   - Current: No insights data
   - Target: Returns `{ prices: { min, max, avg, last }, recommendations: [{ id, name, last_price, frequency }] }`
   - Acceptance: Calling `/api/artikel/5/insights` returns price stats from dokument_positionen table and top 3 co-occurring articles

6. **Co-Occurrence Recommendations**: When an existing article is selected, call `/insights` immediately and show "Oft zusammen verwendet" recommendations at the bottom of the modal.
   - Current: No recommendations in article search
   - Target: On existing article select, recommendations section populates instantly
   - Acceptance: Selecting "Kupferblech" shows "Rinnenhalter Zink" as a co-purchased item if they appear together in dokument_positionen

7. **Pre-Fetch Recommendations for New Articles**: After a new article is saved, call `/api/warengruppen/{id}/recommendations` to show similar articles as proxy recommendations.
   - Current: New articles show no recommendations
   - Target: After creation, recommendations from the same Warengruppe are shown
   - Acceptance: New article saved with "Material" → shows top articles in Material category as recommendations

8. **Toggle Mode Category Filtering**: Clicking a Warengruppe chip filters the search results to that category. Typing then searches within that category.
   - Current: No category filtering in article search
   - Target: Click "Werkzeug" chip → only Werkzeug articles shown → typing filters within Werkzeug
   - Acceptance: With "Werkzeug" chip active, searching "Schraube" only returns Werkzeug articles containing "Schraube"

9. **NumberingService for ART and KD**: Extend NumberingService to generate unique `artnr` and `kundnr` for quick-added records.
   - Current: NumberingService only handles document types (RE, AN, etc.)
   - Target: Generates `ART-2026-XXXX` and `KD-2026-XXXX`
   - Acceptance: Quick-adding an article generates `ART-2026-0001` as its artnr

## Boundaries

**In scope:**
- `SmartCombobox.tsx` — generic, reusable for both Kunden and Artikel
- `QuickAddModal.tsx` — article search + create modal with Warengruppe chips and recommendations
- Updated `QuoteCreationPage.tsx` using new components
- Updated `CustomerSearchCombobox.tsx` with Smart Check and Quick-Add
- Backend: `/warengruppen/simple` endpoint
- Backend: `/kunden/quick-add` and `/artikel/quick-add` endpoints
- Backend: `/api/artikel/{id}/insights` endpoint
- Backend: `/api/warengruppen/{id}/recommendations` endpoint
- NumberingService extension for ART and KD types
- 300ms debounce on all search inputs

**Out of scope:**
- Editing Warengruppen from the modal (separate page)
- Bulk import of articles/customers
- Price negotiation or discount rules
- Multi-language article descriptions
- Inventory/stock management

## Constraints

- **Tech Stack**: Must use existing project libraries (Heroicons, Headless UI, Axios). No new UI library installations.
- **Performance**: `/insights` must return in <200ms for SQLite. If slow, add caching layer.
- **No Duplicates**: All Quick-Add operations must pass a duplicate check before saving.
- **Default Warengruppe**: If no Warengruppe is selected, do NOT save — user must pick one (enforced in frontend).

## Acceptance Criteria

- [ ] 300ms debounce on article and customer search inputs
- [ ] Duplicate customer creation shows warning dialog before saving
- [ ] Article Quick-Add shows Warengruppe chips (Material, Werkzeug, Dienstleistung, etc.)
- [ ] Saving new article requires Warengruppe selection (button disabled until selected)
- [ ] "+ {search_term}" button pre-fills with the user's typed search text
- [ ] Selecting an existing article immediately shows recommendations from `/insights`
- [ ] New articles show proxy recommendations from same Warengruppe after creation
- [ ] Category chips filter search results (Toggle Mode: chip → category, then search within)
- [ ] Quick-added articles get unique `ART-2026-XXXX` artnr
- [ ] Quick-added customers get unique `KD-2026-XXXX` kundnr
- [ ] `/api/artikel/{id}/insights` returns correct min/max/avg/last prices from dokument_positionen
- [ ] Recommendations show top 3 co-occurring articles by frequency
